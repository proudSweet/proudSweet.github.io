写在前面

大概两年前，开了个博客，然无疾而终。前些日子，开了个微信公众号，一直没有发文。

至于为什么之前一直没能坚持写博客，我想有两点吧。

  • 一是自己更倾向于走 精品 路线，所以需要更多的精力与更强的知识储备，而这都是不容易的。
  • 二嘛，自然是因为 懒…

最近一直想把博客和公众号弄起来。趁着今天休息，重新建了个博客，还特地买了个域名（花了钱就要努力更新呀，哈哈），折腾半天，总算可以上线了。

这几年踩过无数坑，越来越觉得 输出 和 总结 是至关重要的，大多数人并非天才，学习新知识的时候，即使能很快接受，但也容易很快忘掉，特别是对于较复杂的内容。然后就要再看，再忘，如果反复几次，方才记牢。遗忘与反复不可避免，但是输出减少了这一过程所需的时间与次数。从我的经验来说，输出 有以下几个帮助：

  • 输出的同时，记得更牢
  • 遗忘某些内容时，可以根据之间的输出笔记、思维导图等，按图索骥，更快的回忆起来
  • 知识本身是零碎的，书籍、公开课、paper、blog, 各种各样的知识来源，我们看的时候，往往觉得掌握了，但没有记录、整理并吸收为自己的知识，依然讲不出这些所谓的你已理解的知识。所以输出的过程，就是内化为自己知识的过程。
  • 阅读的时候，身在局部，自然不知全貌。总结输出时，会对某个topic/domain/research有更全面、清晰的认识。站在更高的角度，自然有更好的视角，不论是掌握知识，还是研究创新，你都会处于更高的level.

以上是这几年学习过程的一些思考，也是促成我重新开博客、公众号的动机。既然有这样的认识，那就坚持去做！

博客标准

说完博客内容，下面谈谈写博客的标准，或者说该怎么去写。

开篇也提过，我喜欢 精品，这是写作的核心标准。围绕这个标准，我想我会这样去组织和建设我的博客。

  • 理论篇

    主要包括

    • NLP
    • ML

    • Math

      偏重于算法，我的标准就是：

    • 全面：当成综述来写

    • 深入：博采众长，深入理解

    • 个人理解：个人总结、思考

      理论篇的写作内容主要会有：

    • 综述

    • 经典算法
    • 各种算法的比较
    • 各种topic / theme

  • 工程篇

    主要包括

    • BigData
    • Programming
    • Project

    • Architecture

      其实工程篇很容易沦为 无病呻吟，好书、其他博客、官方文档 多的是，不要做这些资料的搬运工，一定要深入，这才是工程类博客所需要的。教别人如何写代码，某个类的用法，如何配置等等都是不可取的，我们完全可以从别的地方找到答案。

      因此，工程类的主要内容有：

    • programming language的重要 topic

    • 编程思想
    • library 架构
    • library 核心原理
    • 工程设计、架构 经验

  • 生活篇

    主要包括

    • Blog

    • Joy

      这个一向擅长，各位不要被我秀得头皮发麻，哈哈。
